# 题解点评标准

点评用户题解时读取本文件。目标是让用户知道“哪里有问题、为什么、怎么改成可迁移的最优模式”。

## 评价维度

- 题意理解：是否抓住目标、输入输出、约束和特殊情况。
- 算法选择：是否命中当前主题核心模式，是否存在明显更优的标准做法。
- 正确性：是否能覆盖正常路径、边界路径和反例。
- 复杂度意识：是否能说清时间复杂度、空间复杂度，以及瓶颈来自哪里。
- 代码清晰度：变量命名、控制流、数据结构选择是否容易复盘。
- 模式迁移：用户是否能把本题抽象成可复用模板或判断信号。

## 问题严重程度

- 阻塞级：会导致错误答案、越界、死循环、未定义行为或无法编译。
- 主要级：大部分样例可过，但复杂度不达标、关键边界缺失或思路不稳定。
- 次要级：命名、结构、重复逻辑或小范围可读性问题。

点评时优先写阻塞级和主要级问题，不要把文档填满琐碎风格建议。

## 最优解法选择

- 选择 LeetCode 可接受且面试中最常见的最优或近最优方案。
- 多个方案复杂度相同的时候，选择最贴合当前周主题、最容易迁移的一种。
- 如果某题理论最优写法非常技巧化，而用户当前阶段更适合掌握次优稳定写法，可以说明取舍，但仍要给出“推荐最优解代码”。
- 讲解最优解时必须包含核心思想、关键不变量、复杂度和容易写错的点。

## 最优解代码要求

- 使用 C++20。
- 使用 LeetCode `class Solution` 提交格式。
- 不包含 `main`、本地打印、文件读写或调试宏。
- 代码应可读，变量名表达含义。
- 只在关键不变量或容易误解处写简短注释。
- 不依赖本仓库 `header.h` 或本地模板。

## 常见反馈方向

- 哈希题：检查 key 是否唯一、频次是否正确维护、是否遗漏负数/重复值。
- 双指针题：说明指针移动依据，避免凭感觉移动。
- 滑动窗口题：写清窗口含义、扩张和收缩条件。
- 栈题：说明栈里保存的是值、下标还是状态。
- 链表题：优先指出空链表、单节点、头节点变化和断链风险。
- 二分题：明确搜索区间、循环条件、答案收缩方向和返回值。
- 树题：区分“节点贡献值”和“全局答案”，说明递归返回含义。
- 回溯题：说明路径、选择列表、终止条件和撤销动作。
- 动态规划题：先定义状态，再写转移，最后处理初始化和遍历顺序。

## 下一步建议选择

从下面动作里选最合适的一到两个：

- 重写当前题：用户核心模式没掌握，或代码有阻塞级问题。
- 补边界测试：思路基本正确，但边界覆盖弱。
- 做跟练题：当前题已过，需要巩固同一模式。
- 进入下一题：当前模式稳定，可以推进当天题单。
