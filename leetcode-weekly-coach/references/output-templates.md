# 输出模板

生成周计划、每日题单、单题点评或进度更新时使用这些模板。整体语气保持简洁、具体、鼓励式。

## 每日题单模板

```md
## 第 X 天 - <当天重点>

今天承接：<上一天/上一题暴露的能力点>。

1. `<题号>. <题目名称>` (`难度`) - <为什么先做它>
2. `<题号>. <题目名称>` (`难度`) - <它训练的核心模式>
3. `<题号>. <题目名称>` (`难度`) - <它作为变体或拔高的价值>

完成后重点复盘：<1-2 个最值得观察的点>。
```

## 单题题解点评模板

````md
# 题解点评 - <题号>. <题目名称>

## 基本信息

- 日期：<YYYY-MM-DD>
- 主题：<主题>
- 难度：<简单 / 中等 / 困难>
- 本次结果：<通过 / 错误答案 / 超时 / 未完成 / 仅有思路>
- 对应源码：<src/...cpp 或 未提供>
- 资料来源：<LeetCode 官方 / 其他来源 / 本地参考 / 未联网校准>
- 查询日期：<YYYY-MM-DD 或 未联网>

## 用户思路概述

<先公平、准确地复述用户做法。不要直接从标准答案开始。>

## 主要问题

- <具体问题 1>：<为什么会错、为什么不够优，或在哪类边界上风险最大>
- <具体问题 2>：<同上>

## 做得好的地方

- <具体优点>
- <具体优点>

## 最优解法

<说明推荐最优解的核心思想、关键不变量，以及为什么它比当前写法更稳或更优。>

- 时间复杂度：`O(...)`
- 空间复杂度：`O(...)`

## 最优解代码

```cpp
class Solution {
public:
    // ...
};
```

## 对比与改进方向

<说明用户解法和最优解之间最关键的差距，指出下一次写代码时要抓住的模式。>

## 下一步建议

<明确建议：重写当前题 / 补边界测试 / 做跟练题 / 进入下一题。>

## 后续练习

1. `<题号>. <题目名称>` - <为什么适合紧接着练>
2. `<题号>. <题目名称>` - <为什么适合作为迁移练习>
````

## 文件命名

除非用户指定其他位置，否则默认使用：

- 周计划：`leetcode-plans/week-XX-<topic>.md`
- 单题点评：`leetcode-reviews/YYYY-MM-DD-<problem-number>-<problem-slug>.md`
- 题号未知：`leetcode-reviews/YYYY-MM-DD-<problem-slug>.md`

## 语言规则

- 标题、说明、评价、建议、总结全部使用简体中文。
- 题目名可以保留 LeetCode 官方英文标题，必要时补一句中文说明。
- 最优解代码使用 C++20 和 LeetCode `class Solution` 提交格式。
- 只有当用户明确要求其他语言时，才切换输出语言。

## 点评语气规则

- 表扬要具体，不要空泛。
- 批评针对思路和实现，不针对人。
- 最重要的问题放前面。
- 即使答案完全错误，也保持支持性和建设性。
